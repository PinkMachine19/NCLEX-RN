Architecture Decisions

Structural decisions documented with reasoning — cloned from react-learn-with-ai philosophy.

ADR-001 — Plain JavaScript (no TypeScript) Pre-curriculum

Decision: Plain JavaScript throughout. No TypeScript.

Why: Same rationale as source project — one mental model at a time. Type errors distract from clinical content.

ADR-002 — Static HTML docs (no build step) Pre-curriculum

Decision: Documentation is static HTML with shared styles.css.

Why: Zero dependencies. Openable anywhere. AI sessions read source directly.

ADR-003 — Domain migration from react-learn-with-ai Pre-curriculum

Decision: Replicate exact folder structure, session flow, quiz engine, and progression gates from react-learn-with-ai. Only educational domain changes.

Why: Proven learning platform architecture. NCLEX-RN content swaps into existing patterns without redesign.

ADR-004 — Mock patient data before live integrations Pre-curriculum

Decision: Patient cases live in /src/mock/ and /src/study/. No EHR integration until later modules.

Why: Same mock-first philosophy as source project. Study independently of external systems.

ADR-005 — 10 modules × 4 sessions = 40 session curriculum Pre-curriculum

Decision: Mirror 40-session structure. 10 NCLEX domains with 4 sessions each. Gate quiz at session 4, 8, 12, etc.

Why: Preserves progression checkpoint rhythm from source project.
